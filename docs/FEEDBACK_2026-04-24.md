### 10. CRM 내 피드백 기능 ⭐ (우선 구현 — Claude 자동 처리 파이프라인)

**핵심 아이디어**: 팀원 ↔ 관리자 ↔ Claude 3자 협업 루프

**워크플로우:**
1. **팀원** — CRM 안에서 버그/기능 요청 등록
2. **관리자(David)** — 해당 요청에 댓글로 "이렇게 구현하면 될 것 같다" 지시 작성
3. **Claude** — 매일 새 관리자 지시사항을 자동 확인 → 판단 → 구현

**DB 스키마 (개발일지 자동 생성 포함):**
```sql
-- 메인 피드백 (요청 → 개발일지 자동 전환)
user_feedbacks (
  id uuid PK,
  title text,
  description text,
  category text,  -- 'bug' | 'feature' | 'improvement' | 'question'
  status text,    -- 'submitted' | 'reviewing' | 'planned' | 'in_progress' | 'done' | 'wont_do'
  priority text,  -- 'high' | 'normal' | 'low'
  target_page text,  -- 관련 페이지 (자동 입력)
  created_by uuid FK→users,

  -- 개발 추적 (이 필드들이 채워지면 = 개발일지)
  planned_at timestamptz,          -- 구현 예정 결정일
  planned_for date,                 -- 예정 완료일
  started_at timestamptz,           -- 작업 시작일
  completed_at timestamptz,         -- 실제 완료일
  resolution_summary text,          -- 어떻게 해결했는지 요약
  pr_urls text[],                   -- 관련 PR 링크 (복수)
  commit_shas text[],               -- 관련 커밋 SHA (복수)
  affected_files text[],            -- 변경된 주요 파일

  created_at timestamptz,
  updated_at timestamptz
)

-- 댓글 + 작업 로그 (timeline 자동 구성)
feedback_comments (
  id uuid PK,
  feedback_id uuid FK,
  author_id uuid FK→users,     -- null이면 Claude
  author_type text,             -- 'user' | 'admin' | 'claude'
  comment text,
  is_admin_directive bool,      -- 관리자 지시사항 표시
  is_claude_report bool,        -- Claude 작업 보고 표시
  claude_processed_at timestamptz,
  created_at timestamptz
)
```

**이게 왜 개발일지가 되는지:**
- `status='done'` + `completed_at IS NOT NULL` 건만 모으면 **시간순 완료 이력**
- `resolution_summary` + `pr_urls` → 개발 내역 상세
- `feedback_comments` timeline → 어떻게 논의되고 구현됐는지 전 과정
- 별도 문서 관리 불필요 — DB 한 곳에서 요청부터 배포까지 전체 추적

**UI:**
- 전역 플로팅 버튼 (우측 하단) — 모든 페이지에서 피드백 등록 가능
- `/feedback` 리스트 페이지 (본인 등록분 / 전체, status 필터)
- `/feedback/[id]` 상세 페이지 (timeline 형식 댓글 스레드)
- 관리자 전용: 댓글 작성 시 **"⚡ Claude 지시사항"** 체크박스 → `is_admin_directive=true`
- `/feedback/changelog` — **개발일지 뷰** (status='done' 만 시간순)
  - 월별/주별 그룹핑
  - resolution_summary + PR 링크 표시
  - 카테고리별 필터 (버그 fix / 새 기능)
- 관리자 대시보드 위젯: "이번주 완료 N건", "대기중 N건"

**Claude 자동화:**
- 매일 오전/저녁 scheduled task (CronCreate)로 Claude session 자동 시작
- 쿼리: `feedback_comments WHERE is_admin_directive=true AND claude_processed_at IS NULL`
- 해당 건 읽고 → 판단 → 코드 구현 or 이슈 제기
- 완료 시 `claude_processed_at` 타임스탬프 기록
- Slack으로 관리자에게 "오늘 처리한 N건" 보고

**우선 구현 이유:**
- 이 기능이 있어야 앞으로 팀 피드백(버그/기능 요청)이 **체계적으로 수집 + 자동 처리**됨
- 다른 요청(#1~#9)은 이 기능으로 등록해서 진행하는 게 효율적
